# Intake Context Protocol

When dispatched, each agent receives the path to the intake context file. Read it to understand:

- **Motivation**: Why the user is researching this topic
- **Prior knowledge**: What the user already knows about the topic
- **Constraints**: Scope limitations, time bounds, or focus areas
- **Desired output**: What format and depth the user expects

Use these four dimensions to tailor your output to the user's specific needs. Connect findings, recommendations, and analysis back to the user's stated motivation and constraints.
